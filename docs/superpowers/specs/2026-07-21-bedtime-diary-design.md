# 취침 전 초간단 일기 앱 — 설계 문서

날짜: 2026-07-21

## 목적
자기 직전에 오늘 있었던 일을 자유롭게 짧게 적고, 내일의 아주 간단한 목표를 적는 모바일 다이어리 앱. 자기 직전 사용을 고려해 눈이 편안한 색상/디자인이 핵심 요구사항이다.

## 결정사항

| 항목 | 결정 |
|---|---|
| 플랫폼 | PWA (웹앱, 홈 화면에 추가). 앱스토어 배포 없음 |
| 저장 | 로그인 없이 기기 로컬(`localStorage`)에만 저장, 클라우드 동기화 없음 |
| 알림 | 없음. 생각날 때만 열어서 쓰는 용도 |
| 입력 길이 | 제한 없이 자유롭게 늘어나는 텍스트박스 |
| 지난 기록 열람 | 필요하지만 UI는 최대한 숨겨져 있어야 함 |
| 지난 기록 접근 방식 | 우측 상단의 아주 작고 흐릿한 점 아이콘 |
| 수정 가능 여부 | 작성 당일에만 수정 가능(자동저장), 날짜가 지나면 자동으로 읽기 전용 잠금 |
| 어제 목표 표시 | 오늘 화면 상단에 어제 적은 "내일 목표"를 참고용으로 조용히 다시 보여줌 |
| 기술 접근 | 빌드 도구/프레임워크 없이 순수 HTML/CSS/JS + PWA manifest + 서비스워커 |

## 비주얼 디자인 (시각 동반 도구로 확정)
**팔레트: 세피아 라이트**
- 배경: `#f1e8d8`
- 본문 텍스트: `#4a4038`
- 라벨/포인트: `#a68a5c`
- 입력창 배경: 검정 4% 오버레이 `rgba(0,0,0,0.04)`
- 카드/입력창 모서리: 둥글게 (10px)

어두운 배경(웜 다크, 세이지 다크)도 검토했으나, 어두운 화면이 부담스러울 수 있다는 이유로 따뜻한 종이 톤의 세피아 라이트를 최종 선택.

**레이아웃: 우측 상단 작은 점 아이콘**
좌우 스와이프, 아래로 당기기 방식도 검토했으나 발견 가능성과 미니멀함의 균형을 고려해 작은 점 아이콘 방식을 채택.

## 데이터 모델
`localStorage`에 날짜별 JSON 객체로 저장 (키: 기기 로컬 기준 `YYYY-MM-DD`).

```json
{
  "2026-07-21": { "entry": "오늘 있었던 일...", "tomorrowGoal": "운동 30분" },
  "2026-07-20": { "entry": "...", "tomorrowGoal": "..." }
}
```

## 화면 흐름
1. **오늘 처음 열었을 때**: 어제 "내일 목표"가 상단에 표시되고, "오늘 하루" 텍스트박스와 "내일 목표" 입력창이 보임. 타이핑 시 디바운스 자동저장 (별도 저장 버튼 없음).
2. **오늘 같은 날 다시 열었을 때**: 저장된 내용을 불러와 계속 수정 가능.
3. **날짜가 지나면**: 이전 기록은 자동으로 읽기 전용이 되고, 새 날짜의 빈 입력 화면이 나타남.
4. **점 아이콘 클릭**: 지난 기록 목록(날짜 + 첫 줄 미리보기) → 항목 클릭 시 해당 날짜 읽기 전용 상세 뷰 → 뒤로가기로 오늘 화면 복귀.

## 파일 구조
- `index.html` — 단일 페이지
- `style.css` — 세피아 팔레트, 카드/입력창 스타일, textarea 자동 높이 조정
- `app.js` — 날짜 계산, localStorage 로드/저장, 잠금 로직, 지난 기록 목록/상세 뷰
- `manifest.json` — 앱 이름, 테마 컬러, `display: standalone`, 인라인 SVG 아이콘
- `sw.js` — 앱 셸 캐싱으로 오프라인 실행 지원

## 빌드하지 않는 것
- 로그인/계정, 서버, 클라우드 동기화
- 푸시 알림
- 작성 당일 이후 기록 수정/삭제 기능
- 태그/검색/통계 등 부가 기능

## 검증 방법
1. `index.html`을 열어 오늘 기록 작성 → 새로고침 후 유지 확인
2. 날짜를 바꿔가며 이전 기록이 읽기 전용으로 잠기는지, 새 날짜에 어제 목표가 표시되는지 확인
3. 점 아이콘 → 지난 기록 목록/상세 보기 동작 확인
4. "홈 화면에 추가" 설치 및 오프라인 실행 확인
5. 확정된 시안과 팔레트/레이아웃 일치 여부 육안 확인
